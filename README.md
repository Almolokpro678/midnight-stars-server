# Midnight Stars Discord Server - 24/7 Website Hosting Guide

## How to Keep Your Website Online 24/7 for Free

### 🌐 Free Static Website Hosting Services

#### 1. **GitHub Pages** (Recommended)
- **Free**: Yes, completely free
- **Uptime**: 99.9%+
- **Custom Domain**: Supported
- **How to deploy**:
  1. Create a GitHub repository
  2. Upload your website files
  3. Go to Settings → Pages
  4. Select source branch (usually main)
  5. Your site will be live at `https://username.github.io/repo-name`

#### 2. **Netlify**
- **Free**: Yes, with generous limits
- **Uptime**: 99.9%+
- **Custom Domain**: Supported
- **Features**: Continuous deployment, SSL, form handling
- **How to deploy**:
  1. Drag and drop your website folder to netlify.com
  2. Or connect to GitHub/GitLab repository
  3. Site goes live instantly with a random URL

#### 3. **Vercel**
- **Free**: Yes, excellent for static sites
- **Uptime**: 99.9%+
- **Custom Domain**: Supported
- **Features**: Global CDN, instant deployments
- **How to deploy**:
  1. Sign up with GitHub
  2. Import your repository
  3. Deploy with one click

#### 4. **Cloudflare Pages**
- **Free**: Yes, very generous free tier
- **Uptime**: 99.9%+
- **Custom Domain**: Supported
- **Features**: Global CDN, DDoS protection
- **How to deploy**:
  1. Connect to GitHub repository
  2. Configure build settings
  3. Deploy globally

#### 5. **Firebase Hosting**
- **Free**: Yes, 10GB storage, 360MB/day transfer
- **Uptime**: 99.9%+
- **Custom Domain**: Supported
- **How to deploy**:
  1. Install Firebase CLI
  2. Run `firebase init hosting`
  3. Deploy with `firebase deploy`

### 🚀 Quick Deployment Steps

#### Option 1: GitHub Pages (Easiest)
1. Create a GitHub account
2. Create a new repository named `midnight-stars-server`
3. Upload all your website files
4. Go to Settings → Pages
5. Select "Deploy from a branch"
6. Choose main branch and save
7. Your site is live!

#### Option 2: Netlify (Drag & Drop)
1. Go to netlify.com
2. Drag your entire Server folder onto the page
3. Your site is instantly live with a random URL
4. You can change the URL later

### 🔧 Advanced Options

#### **Custom Domain Setup**
1. Buy a domain (Namecheap, GoDaddy, etc.)
2. Point DNS to hosting provider
3. Configure SSL certificate (usually automatic)

#### **Performance Optimization**
- Enable Gzip compression
- Use CDN (most providers include this)
- Optimize images and assets
- Minify CSS/JS files

### 💡 Pro Tips for 24/7 Uptime

- **Use multiple hosting providers** for redundancy
- **Set up monitoring** with Uptime Robot (free)
- **Enable automatic deployments** from GitHub
- **Use a CDN** for faster global access
- **Regular backups** of your website files
